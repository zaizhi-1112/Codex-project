# AI新闻助手

基于你提供的原型实现的单页新闻助手，支持：

- 多信源抓取（默认内置 AI 官方/媒体 + 中文信源，含 IT之家）
- 点击 `生成最新新闻` 自动执行：48 小时过滤、严格相关性筛选、去重整合、公司多样性约束、50 条内输出
- 排除项过滤（含“扫地机器人”“合作伙伴招募”等）
- 右侧结果可拖拽排序、手动编辑、复制、保存
- 可在“提示词设置”里调整信源、优先词、排除词
- 可在“提示词设置 -> 模型配置”里配置 `Base URL / API Key / Model ID` 并切换运行模式
- 左侧候选标题与右侧精选结果默认自动中文化（先判断中文，必要时自动翻译）
- 支持从左侧列表手动勾选加入右侧，或一键从左侧筛选50条优质新闻
- 已保存旧配置时，启动后会自动补齐内置中文信源（包含 IT 之家）

## 文件结构

- `D:\自己的项目\新闻助手\index.html`
- `D:\自己的项目\新闻助手\styles.css`
- `D:\自己的项目\新闻助手\app.js`

## 启动方式

优先用本地静态服务器打开（避免浏览器对本地文件的跨域限制）：

1. 如果你有 Node.js：
   `npx serve "D:\自己的项目\新闻助手"`
2. 如果你有 Python：
   `python -m http.server 8080 --directory "D:\自己的项目\新闻助手"`
3. 然后在浏览器打开命令输出的本地地址（例如 `http://localhost:8080`）。

## 使用说明

1. 点击 `提示词设置`，可编辑信源（每行：`名称|URL|权重`）。
2. 在“模型配置”里可选三种模式：
   - `仅规则筛选（无需模型）`
   - `模型辅助筛选/总结`
   - `模型联网搜索 + 总结`
3. 如果使用模型模式，配置：
   - `Base URL`（如 `https://api.openai.com/v1`）
   - `API Key`
   - `Model ID`（如 `gpt-4.1`、`qwen-plus-latest`）
4. 点击 `生成最新新闻` 开始抓取并筛选。
5. 左侧显示候选新闻；右侧显示精选结果。
6. 右侧可拖动调整顺序，复制时会按 `1. 2. 3.` 列表格式输出。
7. 左侧支持：
   - `勾选加入精选`：先勾选左侧新闻，再加入右侧
   - `一键筛选50条`：直接从左侧候选里筛选优质50条到右侧

## 联网搜索需要的模型能力

- 重点不是“模型名字”，而是“接口是否支持联网搜索工具”。
- 可用类型：
  - 支持 `Responses + web_search` 的 OpenAI 兼容平台
  - 或支持 `enable_search` 一类参数的平台（例如部分 Qwen 接口）
- 仅有普通聊天模型（例如只会对话、没有搜索工具）时：
  - 可以做“模型辅助筛选/总结”
  - 不能保证真正联网检索新网页

## 说明

- 抓取采用“直连 + 代理回退”模式（`allorigins` / `r.jina.ai` 的 `http` + `https` 回退）提升可用性。
- 若个别信源不可达，不影响其他信源继续生成。
- API Key 仅保存在浏览器本地存储，请在可信环境使用。
